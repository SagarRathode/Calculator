# Architecture

The calculator is divided into multiple layers:

```text
User Input
    ↓
Tokenizer (Lexer)
    ↓
Parser
    ↓
Evaluator
    ↓
Result
```

This separation makes the project:

- easier to understand,
- easier to debug,
- easier to extend,
- closer to real-world software systems.

---

# Tokenizer

The tokenizer converts raw text into tokens.

Example:

```text
(4 + 5) * 6
```

becomes:

```text
LEFT_PAREN
NUMBER(4)
PLUS
NUMBER(5)
RIGHT_PAREN
MULTIPLY
NUMBER(6)
```

---

# Parser

The parser uses **Recursive Descent Parsing**.

Grammar:

```text
expression -> term ((+ | -) term)*
term       -> factor ((* | /) factor)*
factor     -> NUMBER | '(' expression ')'
```

This naturally handles operator precedence.

Example:

```text
2 + 3 * 4
```

is interpreted as:

```text
2 + (3 * 4)
```

```bash

```
